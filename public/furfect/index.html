How to Scope Requests to Your Indian Dev Team

The reason Web Skitters overcharges is because vague requests let them inflate scope. Here's how to write requests that lock them into a fixed price:

  • Be specific: "Change the background color of the swipe card from #FFFFFF to #F5F5F5" not "make the cards look better"
  • Include screenshots: Circle exactly what you want changed. Annotate with arrows. Remove ambiguity.
  • Set a time cap: "This should take no more than 2 hours. Please confirm before starting if you estimate more."
  • Request itemized quotes: "Break down the cost per change, not one lump sum." This prevents them from bundling a $50 fix with a $500 feature.
  • Get the code first: Ask them to push the latest codebase to a GitHub repo you own. If they resist, that's a red flag. Your code is your asset.

App Security Checklist You Can Run Today

You don't need me to check the basics. Open your app right now and verify these:

  • HTTPS: Is your app and website served over HTTPS? If the URL says HTTP, your users' data is being sent in plain text.
  • API keys in code: Search your codebase for any hardcoded API keys, passwords, or secrets. If they're in the frontend code, anyone can see them.
  • User input validation: Try typing <script>alert('test')</script> into any text field in your app. If an alert pops up, you have an XSS vulnerability.
  • Error messages: When something breaks, does the app show a technical error with file paths or database info? That's leaking your architecture to attackers.
  • Password storage: Ask your dev team: "Are passwords hashed or stored in plain text?" If they don't know or say plain text, that's a critical issue.
  • Data access: Can you access other users' data by changing the URL or ID in an API call? Try it. If yes, you have a broken access control issue.

How to Automate Shelter Outreach Without Sounding Like AI

You can send 50 shelter emails a day without writing each one by hand and without sounding robotic. Here's the framework:

  • Step 1: Build a shelter list. Go to Petfinder.com or AdoptAPet.com. Search your target area. Every result has a shelter name, location, and contact. Put them in a Google Sheet: Name, Email, City, Number of Dogs Listed.
  • Step 2: Write 3 templates, not 1. AI-generated emails get caught when every email is identical. Write three versions of the same email with different openings:
    Version A: "I saw you have [number] dogs listed on Petfinder..."
    Version B: "I'm reaching out to shelters in [city] that are doing great work..."
    Version C: "A friend adopted from [shelter name] and told me about the work you do..."
  • Step 3: Personalize the first line. This is the only line that matters. Spend 30 seconds on each shelter's website or social media and write one specific thing: "I saw your post about Max the pit bull mix who's been waiting 6 months." That one line makes the entire email feel handwritten.
  • Step 4: Use mail merge, not AI. Google Sheets + a free tool like Yet Another Mail Merge (YAMM) or GMass. It pulls [Name], [City], [Specific Detail] from your spreadsheet into the template. Each email is unique because the variables are different, but you're not writing 50 emails.
  • Step 5: Send 10-15 per day, not 50. Gmail flags bulk sends. Keep it under 15/day from one account. If you want more volume, use a second email address and alternate days.
  • Step 6: Follow up on Day 3. Most shelters won't reply to the first email. Send a short follow-up: "Just bumping this up. Would love to list your dogs on Furfect Match for free. No cost on your end." Keep it to two sentences. The follow-up gets more replies than the original.

The rule: AI writes the structure. You write the first line. That's what makes it feel human. If the first line could be sent to any shelter, rewrite it. If it could only be sent to that specific shelter, send it.
